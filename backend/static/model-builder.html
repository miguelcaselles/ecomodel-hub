Define Parameters

Add all the input parameters that users will configure for their scenarios

Parameter Categories:
  • Costs: Drug costs, administration costs, state-specific costs (e.g., cost_drug_a, cost_state_stable)
  • Probabilities: Transition probabilities, event risks (e.g., prob_s_to_p_a, mortality_rate)
  • Utilities: Quality of life weights (0-1 scale) for each health state (e.g., utility_stable, utility_progression)
  • Time: Time horizon, cycle length, discount rates (e.g., time_horizon, discount_rate)

Tip: Use descriptive names like "cost_drug_a" instead of "c1"

Typical Markov Model Parameters:
  • Time horizon (5-20 years for chronic diseases)
  • Discount rate (typically 3% for costs and QALYs)
  • Drug costs for each treatment arm
  • State-specific costs (stable, progression, death)
  • Transition probabilities between states
  • Utility values for each health state (0 = death, 1 = perfect health)
